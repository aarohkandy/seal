# Source Inventory

Raw PDFs are present locally in `docs/source-pdfs/` and extracted text is present locally in `docs/extracted/`. Both are intentionally ignored by Git because the exports include private applicant/member data, internal links, resumes, transcripts, email addresses, phone numbers, scores, and operational notes.

## Public-Safe Source Use

Use the PDFs as private research inputs only. Public website copy should be rewritten from sanitized themes, verified project names, and approved images/content.

## Reusable Public Themes

- SEAL stands for Sensors, Energy, and Automation Laboratory.
- Core work spans sensors, energy systems, automation, embedded systems, applied AI/software, sustainability, aerospace/propulsion, biomedical/PPG, water/environment, technical writing, grants, and commercialization.
- The applicant experience should be explained as a simple pathway: explore tracks, apply, onboard, join a project, build a portfolio.
- Existing internal terms such as quests, battlestation, Get Green, HP, rank, dashboards, Clan Life, and KSS should be hidden from public navigation or translated into normal language.
- Public copy should emphasize projects, mentorship, skills, outcomes, and credible research rather than internal scoring mechanics.
